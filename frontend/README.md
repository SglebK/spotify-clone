# Spotify-Clone - Фронтенд

Здесь находится клиентская часть проекта Spotify Clone, написанная на React + Vite. Фронтенд общается с бэкендом, который по умолчанию запущен на `http://localhost:5000`.

## 🔧 Что нужно для запуска

- Node.js 18+ (любая современная LTS версия)
- npm (устанавливается вместе с Node.js)
- Запущенный бэкенд (инструкции в `../backend/README.md`)

## 🚀 Как запустить

1. **Установить зависимости**
   ```bash
   cd frontend
   npm install
   ```

2. **Настроить URL бэкенда (если нужно)**
   - Переменные окружения берутся из файла `.env`
   - По умолчанию `VITE_API_URL = http://localhost:5000` для разработки
   - Для продакшена укажите URL вашего развернутого бэкенда

3. **Запустить в режиме разработки**
   ```bash
   npm run dev
   ```
   Приложение откроется по адресу `http://localhost:5173`

## 🛠 Полезные команды

| Команда | Что делает |
|---------|------------|
| `npm run dev` | Запускает сервер разработки с автоперезагрузкой |
| `npm run build` | Собирает оптимизированную версию для продакшена в папку `dist/` |
| `npm run preview` | Позволяет посмотреть, как будет работать собранный проект |
| `npm run lint` | Проверяет код на ошибки и соответствие правилам ESLint |

## 🧩 Как фронтенд общается с бэкендом

Все запросы к API используют константу `API_URL` из файла `src/components/utils/api/config.js`. Она берёт значение из переменной окружения `VITE_API_URL`, а если её нет — подставляет текущий хост.

Это позволяет коду одинаково работать и на локальной машине, и на сервере. Если вдруг встретите захардкоженный `http://localhost:5000` — смело меняйте на `API_URL` (в последней версии все такие места уже исправлены).

## ✅ Что уже работает

- **Авторизация** — регистрация, вход по JWT токенам
- **Треки** — просмотр всех треков, загрузка своих, прослушивание
- **Плейлисты** — создание, переименование, настройка приватности, удаление, добавление треков
- **Интерфейс** — базовый UI с плеером

## ⚠ Известные проблемы и исправления

- Раньше URL бэкенда был жёстко прописан в компонентах — теперь всё гибко через `API_URL`
- Других критических проблем не обнаружено, проект успешно компилируется и запускается
