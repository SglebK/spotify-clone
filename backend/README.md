# Spotify Clone - Бэкенд

Здесь находится серверная часть проекта Spotify Clone, написанная на Express с использованием Prisma. Фронтенд общается с бэкендом через REST API по адресу `/api`.

## Что сделано / исправлено

- **Аутентификация** через JWT токены (access + refresh)
  - Полный набор endpoints: регистрация, вход, обновление токена, выход, профиль
  - Пароли надёжно хешируются с помощью `bcrypt`
  - Refresh токены сохраняются в базе данных (можно отозвать при выходе)
- **Работа с треками**
  - Просмотр всех треков (`GET /api/tracks`)
  - Просмотр только своих загруженных треков (`GET /api/tracks/my`)
  - Загрузка новых треков с файлами (аудио + опционально обложка) через `multer`
  - Загруженные файлы доступны по ссылкам через папку `/uploads`
- **Плейлисты**
  - Просмотр своих плейлистов (`GET /api/playlists/my`)
  - Создание, редактирование (название, приватность), удаление плейлистов
  - Просмотр конкретного плейлиста со списком треков (`GET /api/playlists/:id`)
  - Добавление треков в плейлисты (`POST /api/playlist-tracks`)
  - Проверка владельца и каскадное удаление связей
- **База данных** полностью переработана
  - Добавлена модель `User` (email, пароль, часовой пояс, refresh токен)
  - Треки и плейлисты привязаны к пользователям
  - Поля приведены в соответствие с ожиданиями фронтенда (`audioUrl`, `coverUrl`, `title`)
  - Создана и применена миграция
- **Настройки и улучшения**
  - Поддержка `.env` файла (порт, данные БД, JWT секреты)
  - Настроены CORS и раздача статических файлов из `/uploads`
  - Улучшена обработка ошибок и валидация

## Как запустить

1. **Установить зависимости**
   ```bash
   cd backend
   npm install
   ```

2. **Настроить окружение**
   - Скопируйте `.env` файл (пример есть в проекте) и укажите свои значения
   - Проверьте, что папки `uploads/audio` и `uploads/covers` существуют (они создаются автоматически)

3. **Применить миграции Prisma**
   ```bash
   npx prisma migrate dev --name init
   npx prisma generate
   ```

4. **Запустить сервер**
   ```bash
   npm run dev    # с автоматической перезагрузкой при изменениях
   # или
   node src/server.js
   ```

Сервер запустится на порту, указанном в `.env` (по умолчанию 5000). Фронтенд может сразу с ним работать без каких-либо изменений.
